# 麻将迷门店复购增长系统｜下一阶段开发边界

## 阶段名称
v0.2.0 可操作业务闭环阶段

## 当前稳定版本
Git tag：v0.1.0-mvp-data-readonly

## 下一阶段目标
从真实数据只读展示系统升级为最小可操作业务闭环系统。

## 核心闭环
玩家创建预约草稿
→ 创建牌局草稿
→ 添加4名参与玩家
→ 保存娱乐积分结果
→ 自动生成战绩海报记录
→ 玩家查看真实战绩海报
→ 领取卡券
→ 老板核销卡券

## 开发原则
1. 一次只做一个最小动作
2. 每个动作单独提交
3. 不一次性做完整业务流
4. 不做登录
5. 不做正式权限
6. 不重新启用 RLS
7. 不引入支付
8. 不做复杂外部依赖

## 当前允许开发
- 创建预约草稿
- 创建牌局草稿
- 添加4名参与玩家
- 保存娱乐积分结果
- 生成 result_card
- 领取卡券
- 核销卡券

## 当前禁止开发
- 现金输赢
- 玩家间结算
- 欠款管理
- 代收代付
- 抽水
- 赌资统计
- 现金排行榜
- 现金奖池
- 正式权限系统
- 正式 RLS 策略
- 支付系统

## 推荐顺序
1. 创建预约草稿提交
2. 创建牌局草稿
3. 添加4名参与玩家
4. 保存娱乐积分结果
5. 生成 result_card 记录
6. 跳转真实战绩海报
7. 卡券领取
8. 卡券核销
9. 统一操作成功/失败提示
10. 重新设计 RLS 权限
11. Vercel 部署

## 每步验收命令
npm run compliance:check
npm run lint
npm run build

## Git 规则
每个最小动作单独提交，不混合提交。
